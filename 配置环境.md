# 配置环境

intel arc a770 16g 使用index-tts2的环境配置。前后倒腾两天快20个小时，尝试了`index-tts2`、`qwen3-tts`和`GPT-SoVITS`三个模型. 其中`qwen3-tts`是安装和使用文档与代码工程整理的最好的, 但效果比不上index-tts, 可能需要调参数好配置, 比较麻烦; `GPT-SoVITS`为了支持N卡的开箱即用, 整理了很多工程代码, 要从源码上修改支持xpu会过于混乱.  

intel显卡使用torch的方式有两种：

1. 在pytorch2.8版本及以前通过ipex（intel-extension-for-pytorch）优化torch =>[官方文档](https://intel.github.io/intel-extension-for-pytorch/xpu/latest/tutorials/getting_started.html) 和 [Github仓库](https://github.com/intel/intel-extension-for-pytorch). 目前已停止维护(archived).
2. 在pytorch2.8版本之后, intel 将ipex和相关优化发布到了pytorch官方项目中, [pytorch文档-Getting Started on Intel GPU](https://docs.pytorch.org/docs/main/notes/get_start_xpu.html), 不太需要单独安装ipex, 只需要安装xpu版本的pytorch, 使用时将`device="cuda"`改为`device="xpu"`即可. 大多数api都已经完成了迁移, 但`torchaudio`还是有兼容性问题, 在我尝试使用torch2.12.x+xpu版本时遇到报错, 通过替换调用库函数完成修改. 但这样合成的语音会有很大的噪音, 不知道是xpu兼容性的问题还是模型本身的问题(issue#512).

采用index-tts2官方项目使用torch2.8版本的xpu版本, ipex优化, 使用conda管理环境而不是uv包管理器.

## conda 与 ipex

```sh
conda create -n xpu python=3.10
conda activate xpu
```

```sh
## 1. 要使用国外的源, zh源已经无法访问403
## 2. 增加额外的源, 处理psutil 包依赖问题
python -m pip install intel-extension-for-pytorch==2.8.10+xpu --index-url https://pytorch-extension.intel.com/release-whl/stable/xpu/us/ --extra-index-url https://pypi.org/simple/
```

```sh
# 测试
python -c "import torch; import intel_extension_for_pytorch as ipex; print(torch.__version__); print(ipex.__version__); [print(f'[{i}]: {torch.xpu.get_device_properties(i)}') for i in range(torch.xpu.device_count())];"

# 若正常, 有警告通知, 表明实现的方式, 是正常的.
```

## index-tts

不使用该项目推荐的uv包管理器，会导致conda环境和uv环境冲突.
删除toml文件中的torch和torchaudio依赖, 上面已经安装了xpu版本.

```sh
# 26年82.x版本的依赖关系查询工具setuptools放弃了就api, 但modelscope依赖该放弃的api, 使用modelscope会报错.解决方案：
# 1. 不使用modelscope, 使用hf
# 2. 降级到81.x版本
pip install "setuptools<82" --force-reinstall

# 之后添加该版本到toml文件的依赖中

# 下载权重文件
modelscope download --model IndexTeam/IndexTTS-2 --local_dir checkpoints

# 检查GPU是否可用. 这里只是检查xpu版本的pytorch是否能正常使用.
python tools/gpu_check.py

# 以可编辑的形式安装为库, 包含[webui]的依赖
pip install -e ".[webui]"
```
